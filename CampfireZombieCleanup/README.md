# CampfireZombieCleanup

Despawns PEAK mushroom zombies that enter an active campfire protection area.

## Behavior

- Finds active `Campfire` objects at runtime.
- Uses each campfire's `moraleBoostRadius`, plus a small configurable padding.
- Removes owned `MushroomZombie` instances that enter the campfire area.
- By default zombies despawn immediately. `KillBeforeDespawn` can be enabled to set the zombie to Dead first, then despawn after a short delay.

In multiplayer, the network owner/host needs the mod installed so zombie removal can be synchronized through PEAK's normal zombie despawn path.

## Configuration

The config file is generated by BepInEx after the first launch.

- `General.Enabled`: enables the cleanup behavior.
- `General.CheckIntervalSeconds`: seconds between scans.
- `Campfire.RadiusPadding`: extra meters added to the campfire radius.
- `Campfire.MinimumRadius`: fallback radius if the game radius cannot be read.
- `Behavior.KillBeforeDespawn`: kills before despawning instead of immediate despawn.
- `Behavior.DespawnDelaySeconds`: delay after killing before despawn.

## Build

```sh
dotnet build -c Release
```

The Thunderstore package is written under `artifacts/thunderstore/` during Release builds.
