# Universal Campfire Fuel

Universal Campfire Fuel changes PEAK campfires so any held item can be offered as fuel before lighting.

## Behavior

- Unlit campfires show the current heat progress and the heat needed to light.
- The required heat scales with living scout count, with a configurable cap so groups do not need excessive fuel.
- Every item receives a deterministic heat value from its item ID/name, with configurable overrides for important items.
- Adding fuel consumes the held item, updates campfire log visuals, and syncs progress through the campfire `SetFireWoodCount` RPC when a Photon view is available.
- Lit campfires keep normal vanilla behavior.

## Configuration

The config file is generated by BepInEx after first launch.

- `General.Enabled`: enables the mod.
- `Heat.BaseRequiredHeat`: heat needed for one scout.
- `Heat.HeatPerExtraScout`: added heat per extra living scout.
- `Heat.MaxRequiredHeat`: cap for the lighting requirement.
- `Items.MinimumItemHeat`: minimum generated heat for unlisted items.
- `Items.GeneratedHeatVariance`: generated heat range above the minimum.
- `Items.HeatOverrides`: comma-separated `itemID=heat` or `name=heat` entries.

Default overrides make firewood and wood-like items valuable while still allowing any item to contribute.
